学OpenGL编3D游戏

第十二章 碰撞检测

加入碰撞后，游戏才有真实的感觉。

    我们介绍两种碰撞情况。漫游戏中的碰撞，子弹与目标的碰撞。

主视点（游戏操作者）与对象(树木)的碰撞绕行。

    我们要求在漫游过程中，当我们遇到树或其它对象后，从它的一边绕行过去，而不象前面那样穿树而过（给游戏加一点智能）。

子弹在飞行过程中击中目标（3D动画模型）产生爆炸的过程。

      子弹击中目标有多种算法。最简单的是直接取瞄准点为爆炸点，这种算法以屏幕定位为基准，无需碰撞检测，但在使用中有一定局限性。这里介绍的算法是基于运动物体碰撞检测的原理：子弹发射后，在飞行途中随时检测有否与什么物体相碰撞。无论什么物体一但与子弹相碰，子弹都爆炸，这样似乎更符合自然规律。 

到此，一个类似"三角洲部队"的简单3D射击游戏就完成了。你玩玩看，游戏中设有20个敌人，看你要多少时间消灭他们。
